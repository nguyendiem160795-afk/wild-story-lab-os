# Purpose

Project Template defines the standard structure for every Wild Story Lab production project.

It ensures that all assets, prompts, documents, metadata, and deliverables are organized consistently, making projects easier to maintain, review, archive, and reproduce.

---

# Objectives

- Standardize project organization
- Improve collaboration
- Simplify production workflows
- Ensure reproducibility
- Support version control
- Enable automation

# Standard Project Structure

```text
Project/
│
├── docs/
├── prompts/
├── assets/
│   ├── characters/
│   ├── environments/
│   ├── props/
│   ├── audio/
│   └── thumbnails/
├── renders/
│   ├── draft/
│   ├── production/
│   └── final/
├── exports/
├── metadata/
├── reports/
└── archive/
```

# Core Principles

## Consistency

Every project follows the same structure.

---

## Traceability

Every output can be traced back to its source assets and prompts.

---

## Version Control

Every major deliverable records its version.

---

## Scalability

The template supports both small and large productions.

---

## Automation Ready

Folder names and metadata follow predictable standards.
