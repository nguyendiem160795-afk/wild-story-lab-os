# EAOSS Master Changelog

> Official change history for the Enterprise AI Operating System Specification (EAOSS).

---

# Purpose (Mục đích)

This document records all significant architectural, governance, and repository-level changes made to the EAOSS specification.

Module-specific changes shall be recorded in each module's own `CHANGELOG.md`.

---

# Scope (Phạm vi)

This changelog covers:

- Repository structure
- Architecture changes
- Documentation standards
- Governance policies
- Naming conventions
- Versioning strategy
- Cross-module decisions

It does **not** include implementation details of individual modules.

---

# Versioning Policy

EAOSS follows **Semantic Versioning**.

| Version Component | Meaning |
|-------------------|---------|
| Major | Breaking architectural or governance changes |
| Minor | New modules, standards, or capabilities |
| Patch | Documentation improvements, corrections, or editorial updates |

Example:

```text
2.1.3
```

**Release Date**

# Contribution Rules

Every repository-level change shall:

- Include a version number.
- Include a release date.
- Describe the reason for the change.
- Be traceable to the affected documents.
- Preserve historical records.

Repository history must never be rewritten.
