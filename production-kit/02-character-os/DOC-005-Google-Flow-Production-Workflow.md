# Purpose

This document defines the official production workflow for creating animated videos in Google Flow.

Every project should follow this workflow to ensure consistency, quality, and production efficiency.

# Project Structure

Each project should contain:

- Character
- Background
- Props
- Scenes
- Prompts
- Final Video

---

# Character Rules

Always use the official Mochi Asset Pack.

Never recreate Mochi from scratch if an approved asset already exists.

---

# Scene Workflow

Each video follows the same order:

1. Hook
2. Introduction
3. Goal
4. Conflict
5. Escalation
6. Solution
7. Ending
8. Call to Action (optional)

---

# Prompt Workflow

Each scene must contain:

- Character
- Action
- Emotion
- Environment
- Camera
- Lighting
- Style
- Audio

---

# Camera Rules

Recommended camera shots:

- Close-up
- Medium Shot
- Full Shot
- Tracking Shot
- Orbit Shot
- Dolly In
- Dolly Out

Avoid excessive camera changes in one scene.

---

# Environment Rules

Backgrounds should remain consistent throughout the story.

Only change environments when the story requires it.

---

# Character Rules

Maintain:

- Face
- Fur color
- Eye color
- Green scarf
- Body ratio
- Personality

Never modify core identity.

---

# Quality Checklist

Before rendering:

- Character consistency verified.
- Prompt reviewed.
- Camera movement smooth.
- Lighting consistent.
- No AI artifacts.
- Child-friendly content.
- Story flows naturally.

---

# Export Settings

Resolution: Highest available

Aspect Ratio:

- 16:9 (YouTube)
- 9:16 (Shorts)

Format:

MP4

---

# File Naming

2026-07-22_Mochi_Cooking

# Version Control

After approval:

- Save project.
- Export final video.

- Update GitHub documentation.
