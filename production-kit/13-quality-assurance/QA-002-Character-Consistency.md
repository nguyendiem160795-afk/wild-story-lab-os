# Purpose

Character Consistency defines the validation standards used to ensure every recurring character remains visually, behaviorally, and narratively consistent throughout all productions.

It establishes objective quality checks before rendering and before publication.

---

# Objectives

- Preserve character identity
- Prevent AI-generated drift
- Maintain brand recognition
- Improve production quality
- Support automated validation

# Golden Reference

Every recurring character must have one approved Golden Reference.

The Golden Reference defines

- Official appearance
- Official outfit
- Official proportions
- Official colors
- Official expressions
- Official personality
- Official voice profile

All generated scenes are validated against this reference.

---

# Drift Detection

Common consistency issues include

- Face changes
- Eye color changes
- Clothing changes
- Fur color variation
- Incorrect accessories
- Incorrect proportions
- Personality inconsistency
- Voice inconsistency

---

# Severity Levels

## Critical

Character is no longer recognizable.

Action

Reject immediately.

---

## Major

Character remains recognizable but differs from the approved reference.

Action

Scene revision required.

---

## Minor

Small visual deviations with no impact on recognition.

Action

Document and monitor.

# QA Checklist

Before approval verify

✓ Golden Reference used

✓ Character Pack version correct

✓ No unauthorized design changes

✓ Consistent proportions

✓ Consistent colors

✓ Consistent expressions

✓ Consistent personality

✓ Character score ≥95%

---

# Best Practices

- Freeze approved character designs.
- Update Character Bible before visual redesigns.
- Use one official Character Pack per production.
- Validate every scene independently.
- Recheck consistency after render generation.
