# KS-002 — Taxonomy

Version: 1.0.0

Status: Production

Owner: Wild Story Lab

Last Updated: 2026-07-23

---

# Purpose

Taxonomy defines the official classification system used throughout Wild Story Lab OS.

It provides a standardized vocabulary and hierarchical organization for all knowledge objects, enabling consistent indexing, search, automation, reporting, and AI reasoning.

---

# Objectives

- Standardize terminology
- Eliminate naming ambiguity
- Improve discoverability
- Support semantic search
- Enable automated classification
- Simplify knowledge governance

---

# Taxonomy Philosophy

Every Knowledge Object belongs to one primary category.

Each category may contain multiple subcategories.

Objects may be tagged across domains but maintain a single canonical classification.

# Naming Consistency

Use controlled vocabulary.

Avoid synonyms for official categories.

Example

Correct

Character Pack

Incorrect

Character Bundle

Character Assets Folder

Hero Pack

---

# Validation Rules

The taxonomy must

✓ Prevent duplicate categories

✓ Maintain hierarchical integrity

✓ Support future expansion

✓ Preserve backward compatibility

---

# Best Practices

- Create new categories only when necessary.
- Prefer reuse over duplication.
- Review taxonomy changes through governance.
- Keep terminology stable.
- Document every taxonomy update.
