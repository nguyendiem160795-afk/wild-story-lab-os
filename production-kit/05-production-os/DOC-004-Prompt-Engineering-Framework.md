# Prompt Engineering Framework

> Official AI prompt engineering framework for Wild Story Lab Production Operating System.

---

# 1. Purpose

This document defines the official prompt engineering framework used by Wild Story Lab.

The purpose is to transform creative production requirements into structured AI instructions.

---

# 2. Prompt Engineering Definition

Prompt Engineering is:

```
The process of designing,

structuring,

and optimizing

instructions for AI generation systems.
```

---

# 3. Prompt Philosophy

Wild Story Lab follows:

```
Clear Instructions

+

Strong References

+

Creative Direction

+

Technical Control

=

High Quality AI Output
```

---

# 4. Prompt Architecture

Every production prompt contains:

```
SUBJECT

+

SCENE

+

CAMERA

+

MOTION

+

STYLE

+

TECHNICAL DIRECTION
```

---

# 5. Subject Definition

The Subject describes the main focus.

Required:

```
Character Identity

Appearance

Expression

Action

Purpose
```

---

Example structure:

```
Character:

Mochi


Appearance:

Approved Character Reference


Action:

Cooking in a classroom
```

---

# 6. Scene Definition

Scene describes the environment and situation.

Include:

```
Location

Time

Atmosphere

Objects

Background Elements
```

---

Example:

```
A bright children's classroom

with colorful toys

and learning decorations.
```

---

# 7. Camera Direction System

Camera direction controls visual storytelling.

Required:

```
Camera Shot

Camera Angle

Camera Movement

Camera Focus
```

---

Common camera shots:

```
Extreme Close Up

Close Up

Medium Shot

Wide Shot

Full Shot
```

---

Camera movements:

```
Dolly In

Dolly Out

Tracking Shot

Orbit Shot

Crane Shot

Pan Movement
```

---

# 8. Motion Description System

Motion defines animation behavior.

Include:

```
Character Movement

Object Movement

Camera Movement

Speed
```

---

Motion examples:

```
Slow emotional movement

Fast playful movement

Smooth cinematic movement
```

---

# 9. Visual Style System

Every prompt must define:

```
Art Style

Lighting

Color

Atmosphere

Rendering Quality
```

---

Example:

```
3D animated cinematic style

soft lighting

high quality rendering
```

---

# 10. Cinematic Direction

Prompts should communicate:

```
Film Language

Visual Emotion

Storytelling Purpose
```

---

Include:

```
Composition

Depth

Perspective

Focus
```

---

# 11. Character Consistency Prompt Rules

Every character prompt must preserve:

```
Face

Body Shape

Fur/Hair

Colors

Clothing

Accessories

Personality
```

---

Never change:

```
Character Identity

Without Version Update
```

---

# 12. Environment Consistency Prompt Rules

Environment prompts must maintain:

```
Location Identity

Design Language

Atmosphere

Visual Style
```

---

# 13. AI Video Prompt Structure

Official format:

```
Camera Shot + Camera Movement

+

Subject Description

+

Scene Description

+

Cinematic Style

+

Lighting

+

Motion Speed

+

Technical Quality
```

---

# 14. Image Prompt Structure

Official format:

```
Subject

+

Environment

+

Visual Style

+

Lighting

+

Composition

+

Quality
```

---

# 15. Prompt Optimization Rules

A strong prompt should be:

```
Specific

Visual

Structured

Consistent

Action Oriented
```

---

Avoid:

```
Vague Description

Conflicting Instructions

Missing Character Details

Unclear Actions
```

---

# 16. Prompt Version Control

Every important prompt requires:

```
Prompt ID

Version

Purpose

Related Scene

Generation Tool
```

---

# 17. Prompt Testing Process

Workflow:

```
Prompt Draft

↓

AI Generation Test

↓

Output Review

↓

Prompt Optimization

↓

Final Prompt
```
